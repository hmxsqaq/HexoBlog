Input System-Unity新输入系统 学习笔记

官方文档：https://docs.unity3d.com/Packages/com.unity.inputsystem@1.5/manual/index.html

Input System的导入

于Unity自带的Windows->Package Manager安装即可，安装后需要重启项目

之后将Project Settings->Player->Others->Configuration->Active Input Handling设置为**Both/Input System Package(New)**以启用Input System

Action概述

Input System通过Action系统实现了输入响应逻辑与用户行为控件的分离，使得对于用户输入的管理与多端设备兼容更加容易

Action的创建

通过脚本创建单一Action

可以通过在继承了Monobehavior的脚本中声明InputAction类型的变量(需要引入UnityEngine.InputSystem命名空间)

通过Inspector/JSON等多种方式对声明的Action进行创建，以下为通过Inspector进行创建:

挂载脚本后便可以在Inspector界面看到声明的moveAction

点击齿轮符号便可对moveAction进行编辑，设置Action的属性

点击加减号可以为Action添加/删除按键绑定，为Action绑定对应按键后，按下按键便可触发与Action相关联的方法

通过InputActionAsset文件统一创建Action

成功安装Input System包之后，于Assets界面右键->Create->Input Actions便可创建Input Actions Asset文件

双击创建出的InputActions即可于专门的界面中统一创建管理Actions

Action的属性

Types

Action共有三种默认的Type，其影响的是Action对于输入值的响应规则

Value Button Pass Through
phase.started 返回值从初始状态被改变时 按钮开始被按下，但尚未超过阈值 第一个控制器的返回值从初始状态被改变时
phase.performed 返回值改变时 按钮被按下超过阈值时 返回值改变时
phase.canceled 返回值不再改变时 按钮被释放 返回值不再改变时

Pass Through与Value基本一致，区别是Value只会响应一个最活跃的输入，而Pass Through会响应所有绑定了这一Action的按键的输入

Interactions

Action共有6种Interactions，其影响的是Action对于输入行为的响应规则

当不设置时会设定为Default，响应与Types相同，其余Interactions理解并不困难，可见官方文档:

https://docs.unity3d.com/Packages/com.unity.inputsystem@1.5/manual/Interactions.html#predefined-interactions

Action与方法的绑定

创建Actions后便可以将Actions与具体的方法进行绑定，方法带有参数<InputAction.CallbackContext>，其为InputSystem的回调参数，返回在Action中获得的用户输入数据

以下是三种绑定方式

直接通过声明的InputAction变量进行绑定

此方法较为直观，简单，无需创建InputSystemAsset文件

但使用较为局限，只能为一个个创建Action，无法统一管理，无法体现出InputSystem的优势

在Inspector界面为MoveAction绑定四个按键为表示二维向量

action中存在started、performed、canceled三个属性，其可以订阅拥有<InputAction.CallbackContext>的方法

通过moveAction.started/performed/canceled += OnMove,便可以在Action运行到指定状态时触发所订阅的方法

PS：务必注意使用Action前需要使用Enable方法启用Action，物体被禁用时也需Disable不需要的Action

通过PlayerInput组件进行绑定

此方法需要用到InputSystemAsset与PlayerInput组件，大部分操作可以通过Inspector界面完成

但提供的四种方法有一定局限性，消息系统效率较差，而Event系统需要在Inspector界面一个个绑定，操作较为繁琐

为指定物体添加Player Input组件，在Action属性中选择创建的Input System Asset文件，便可以指定Asset文件中创建的Action

通过此方法绑定时，共有4种方式可供选择：发送消息、广播消息、Unity事件、C#事件

以下为Unity事件的应用方式：

在脚本中创建公开的参数为<InputAction.CallbackContext>的方法，在物体上挂载该脚本

在PlayerInput->Events对应Action中选择挂载了脚本的物体，选择方法即可

通过InputAction配置文件生成C#类进行绑定

此方法与方法一没有本质区别，但是可以通过自建的InputSystemAsset统一创建管理Action

选择创建的InputActionAsset文件，在Inspector界面中勾选Generate C# Class选项，点击Apply,即可生成与InputSystemAsset同名的C#文件

之后便可以在脚本中new一个生成出的InputAction类出来，其中可以访问InputSystemAsset中创建的所有Action Map与Action

PS：务必注意使用Action前需要使用Enable方法启用Action，物体被禁用时也需Disable不需要的Action
